Шаг 2 · Кадры

Архитектура полей (что в ответе модели и что уходит в генерацию)

Шаг 2 — два вызова чата: 2a раскадровка (markdown по ### Кадр N); 2b классификатор prev по тому же тексту сна, шагу 1 и сырой раскадровке — один JSON с флагами по индексам кадров.

В ответе 2a в основном английские имена полей; ниже — как они отображаются в карточке и как используются дальше.

  • Персонажи — character_references: id людей из шага 1. В image API подмешиваются их листы-референсы (лицо, фигура). Если пусто — пайплайн может вывести id по словам в кадре или перенести список с предыдущей карточки; метки «авто», «с пред. кадра», «+dreamer» в UI показывают это (дубликаты одного человека схлопываются).
  • Окружение — в данных: текст Опора и/или base_reference (id окружения из шага 1). Если кадр идёт с prev, пластина окружения в запрос не подмешивается — в карточке в этой строке показывается — (prev описан в колонке Prev).
  • Prev — решает шаг 2b (поле use_previous_frame на карточке после парсинга JSON). В UI колонка «Prev» — итог для image API по scene continuity (та же сцена/камера без разрыва). Метка 2b у кадра 2+ — сырое решение классификатора.
  • Кадр — итоговое описание кадра (frame_text / Кадр); уходит в image API. Ниже тем же блоком показывается изменение такта (delta / Изменение) — оно тоже входит в текст запроса к image API. К основному тексту сна в запрос добавляется сжатый отрывок для атмосферы и деталей (см. шаг 3, отладка img_prompt).
