# Port Allocation Map

Дата: 2026-05-07. Источники: `ss -tlnp` на хосте, `docker ps`, [`RUNTIME_SOURCE_OF_TRUTH.md`](/home/david/projects/RUNTIME_SOURCE_OF_TRUTH.md). **Ничего не менялось.**

---

## 1. Занятые и зарезервированные порты (production-related)

| Порт | Интерфейс | Служба / назначение |
|------|-----------|---------------------|
| **80** | `0.0.0.0` / `::` | **system nginx** (HTTP) |
| **443** | `0.0.0.0` / `::` | **system nginx** (HTTPS, voice vhost) |
| **22** | все интерфейсы | SSH |
| **3001** | `127.0.0.1` | **agent** (Next в Docker → внутренний 3000) — **корень сайта** через nginx |
| **8002** | `127.0.0.1` | **content** (FastAPI) — `/content`, `/ui`, `/dev`, `/telegram` через nginx |
| **53** | resolv / systemd | DNS stub (не трогать) |

**MongoDB:** в активном `docker ps` порт **27017 не опубликован** на `0.0.0.0` хоста; доступ только из Docker-сети `david_default` (и с хоста при необходимости через `docker exec`, не через открытый порт).

---

## 2. Прочие слушатели на снимке

На **`127.0.0.1`** обнаружены дополнительные порты с процессами **node** (высокие номера: ~35k, ~41k, ~43k). Вероятно **IDE / Cursor / вспомогательные tooling** — перед выбором порта для нового фронта выполнять **`ss -tlnp`** и не полагаться на статический список.

---

## 3. VAPI / WebRTC

Не используют фиксированный локальный порт на сервере в классическом смысле: браузер инициирует сессии к **внешним** сервисам VAPI по **WSS/HTTPS**. Учитывать сетевые политики и CORS при смене origin для `agent-next`.

---

## 4. Туннели (dev)

Скрипты dev могут поднимать **cloudflared** на порт, заданный **`PORT`** в `projects/content/.env` (часто **8000** при локальном `run_dev.sh`). Это **отдельный** сценарий от Docker production (**8002**). Не запускать одновременно без понимания конфликта портов.

---

## 5. Рекомендуемый «безопасный» диапазон для `agent-next` и экспериментов

Цель: **не пересечься** с **3001** и **8002** и с системными **80/443**.

| Назначение | Рекомендуемые порты (кандидаты) | Примечание |
|------------|----------------------------------|------------|
| **`agent-next` dev (`pnpm dev`)** | **3002**, **3003**, **3100** | Перед стартом: `ss -tlnp \| grep LISTEN` |
| **Experimental static/vite** | **5174**, **4174** | Избегать **5173**, если занят другим Vite |
| **Локальный WebSocket / signaling debug** | **3210**, **3211** | Не публиковать наружу без firewall |
| **Procedural engine локальный HTTP (future)** | **8787**, **8790** | Зарезервировать под отладку; не 8080 (часто занят) |

**Рекомендованный зарезервированный блок для экспериментов:** **3010–3019** (next.js альтернативы) и **3220–3229** (WS/отладка) — при условии проверки `ss` перед первым запуском.

---

## 6. Что не делать

- Не вешать экспериментальный фронт на **80/443** без смены system nginx (запрещено текущей политикой без отдельного approval).
- Не использовать **3001/8002** для второго Next-проекта на той же машине.
- Не открывать **27017** на `0.0.0.0` для «удобства» экспериментов.
