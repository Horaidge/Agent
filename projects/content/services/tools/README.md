# Model Tools

Эта папка — единая точка для инструментов модели.

## Структура

- `model_tools/` — **модуль на каждый tool**:
  - `*_tool.py` содержит:
    - имя инструмента,
    - описание для модели,
    - OpenAI schema,
    - парсинг аргументов,
    - (опционально) функцию-исполнитель.
- `openai_definitions.py` — реестр:
  - `OPENAI_TOOLS_DEFAULT` — tools, которые реально передаются в `chat_completion`.
  - `OPENAI_TOOLS_CATALOG` — все известные tools (включая отключённые).
- `image_tools.py`, `video_tools.py` — низкоуровневые Python-реализации.

## Как добавить новый tool

1. Создать файл в `model_tools/`, например `search_memory_tool.py`.
2. Описать:
   - `TOOL_NAME`,
   - `TOOL_DESCRIPTION`,
   - `OPENAI_TOOL_SCHEMA`.
3. Добавить парсер аргументов и исполнение (если нужно).
4. Подключить схему в `openai_definitions.py`.
5. Добавить обработку вызова в `chat_orchestrator.py`.

## Dream Pipeline как единый tool

`model_tools/dream_pipeline_tool.py` — это композитный инструмент
`generate_dream_pipeline` для случая «из текста сна в итоговый ролик».

Он описывает единый вход для модели, но внутри подразумевает 4 внутренних шага:
1. декомпозиция сна;
2. генерация изображений;
3. анимация;
4. склейка финального видео.

## Принцип

Модель видит только schema и описание.
Сервер отвечает за валидацию аргументов, исполнение и безопасный ответ пользователю.
