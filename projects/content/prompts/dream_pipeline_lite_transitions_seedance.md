Ты строишь **JSON-план монтажа** для Dream Pipeline Lite (preset Seedance / i2v).

## Цель

- Сохранить смысл и непрерывность **там, где это уместно одним клипом** (меньше изолированных i2v, дешевле по токенам, плавнее движение).
- Работай от keyframes: **один сегмент на каждый промежуток между соседними keyframes**.
- Не делай дробление на scene 0/2/… по соседним индексам, если это один keyframe-gap.
- Не ломай границы `hard_cut`: смена локации, новый такт, обрыв prev-цепочки, явный «удар» кадра.
- **Сюжетно важные** кадры не «теряй»: каждый такой кадр должен быть **концом или началом** хотя бы одного `animate_transition` (или отдельный «якорный» клип).

## Вход (user)

JSON с `dream_text`, `environments`, `characters`, `frames` (тексты кадров, `use_previous_frame_resolved`, `image_generated_ok` — **без** пикселей и длинных URL). Учитывай системные подсказки: `prompt_mode` (first_last_frame / first_frame_only / text_only), `audio_required`.

## Режимы плана (выбери один и придерживайся)

## Режим плана

1. Добавь `keyframes`: строго возрастающий список индексов `0..N-1`, обязательно включая `0` и `N-1`.
2. В `transitions` добавляй только `animate_transition` для пар соседних keyframes:
   - `(k[0], k[1])`, `(k[1], k[2])`, ...
3. Для каждого сегмента обязательно:
   - `motion_prompt`
   - `segment_mode` (`pairwise` по умолчанию)
   - `duration_sec` (выбери длительность под объём анимации сегмента)
   - `segment_story` (кратко: что происходит между этими keyframes)
   - В `motion_prompt` и/или `segment_story` обязательно укажи фазность: `Early phase`, `Mid phase`, `Late phase`.
4. Если `audio_required=true`, добавляй `voiceover_text` для каждого сегмента (простая связная озвучка по сюжету, без избыточной импровизации).

## hard_cut

В этом пресете `hard_cut` используй только для действительно резкого разрыва, который нельзя выразить keyframe-gap сегментом.

## motion_prompt (только `animate_transition`)

- **1–3 коротких предложения** (ориентир до ~60–80 слов, не длинные абзацы).
- Киноязык: кто/что движется, камера, темп, **что меняется** за короткий клип между опорными кадрами.
- **Не** вставляй длинные цитаты `kad` и **не** пересказывай весь сон; не дублируй поля кадра дословно.
- Учитывай `prompt_mode` и `audio_required` (без титров/реплик в кадре, если `audio_required`).
- Критично: не прыгай напрямую из стартового состояния к финальному keyframe. Опиши микро-акт целиком: `setup -> development -> payoff`.
- Если объект появляется только ближе к следующему keyframe (например, вертолёт), не вводи его в Early phase — появление должно быть в Late phase.
- Разрешён **короткий функциональный диалог** внутри действия (например: просьба -> обсуждение -> выбор/оплата), если он важен для визуального перехода.
- Диалог должен быть сценическим и коротким; без длинных монологов и без литературного пересказа сна.

## Что запрещено в финальном motion_prompt

- Служебные строки и метки контракта: `Micro-act rule`, `Сюжет сегмента:`, отдельные тех-подписи `Early phase: ...` / `Mid phase: ...` / `Late phase: ...`.
- Англоязычные/технические хвосты-инструкции для валидатора вместо режиссёрского текста.
- Глобальный пересказ `dream_text`, который не относится к текущему keyframe-gap.

## Выход

Только **один JSON-объект** (без markdown, без ```), с ключами минимум:

- `scenes` — массив блоков (можно с `scene_summary`).
- `transitions` — массив объектов с полями `from_frame_index`, `to_frame_index`, `transition_type=animate_transition`, `motion_prompt`, `segment_mode`, `duration_sec`, `segment_story`.
- Если `audio_required=true`, включай `voiceover_text`.
- Обязательно добавь `keyframes` (массив индексов).

Допускается `montage_mode` не указывать — сервер выставит сам.
